# 前端面试问题记录

## 1. 缓存策略

### HTML 文件

- **不建议长时间强缓存**：HTML 内容频繁变更（结构、文本、链接）
- **推荐策略**：设置较短缓存时间（几分钟到几小时），确保用户获取最新内容
- **原因**：HTML 通常不带哈希值，强缓存会导致用户看到过时内容

### CSS / JS 文件

- **长时间强缓存**：文件相对稳定，可设置一年缓存时间
- **哈希值版本控制**：构建工具（Webpack、Vite）生成带哈希的文件名，如 `style.abc123.css`
- **原理**：内容变化 → 哈希变化 → 新文件名 → 浏览器获取最新版本

## 2. CORS 同源策略

- **定义**：浏览器保护用户的安全策略，限制跨域请求
- **关键点**：
  - 只有浏览器才有跨域问题
  - Postman 不是浏览器，不执行同源策略
  - 服务器之间请求无跨域限制

## 3. 组件设计

### 公共组件

- **特点**：不处理数据，纯展示
- **核心**：独立、复用
- **示例**：Button、Modal、Table

### 业务组件

- **特点**：服务于特定业务，由多个公共组件组合
- **核心**：组合、实现（业务模块化）
- **示例**：用户管理模块、订单列表

## 6. 网络相关

### Cookie

- 请求自动携带 Cookie
- **限制**：必须是同一域名下的 Cookie

### 表单提交 vs Fetch 提交

| 特性     | 表单提交 | Fetch 提交       |
| -------- | -------- | ---------------- |
| 页面刷新 | 会刷新   | 不刷新           |
| 异步     | 否       | 是               |
| 文件上传 | 支持     | 支持（FormData） |
| 进度监控 | 有限     | 支持 Progress    |

## 7. 进阶问题

### 反射判断 ref 内部复杂类型做 reactive 代理

- 使用 `Reflect` API 判断对象类型
- 复杂类型（Object、Array）使用 Proxy 做响应式代理
- 基础类型使用 `ref`，复杂类型使用 `reactive`

### DevTools 调试

- [Chrome Network 参考文档](https://developer.chrome.com/docs/devtools/network/reference?utm_source=devtools&hl=zh-cn#provisional-headers)
- 用于分析网络请求、缓存策略、性能瓶颈

### Block-level Diffing（块级 Diff）

- **概念**：Vue 3 的优化策略，将模板按动态/静态节点分块
- **原理**：只对比动态节点块，跳过静态节点，减少比较次数
- **优势**：比传统 Virtual DOM 的节点级 Diff 性能更高

### Streaming Tokenizer（流式分词器）

- **概念**：边接收数据边解析，无需等待完整响应
- **应用**：大模型流式输出、视频流处理、大文件读取
- **优势**：降低首字节等待时间，提升用户体验

## 8. MCP vs Skill（AI 开发概念）

| 概念      | 作用                     | 示例                       |
| --------- | ------------------------ | -------------------------- |
| **MCP**   | 给大模型提供**数据**     | 查询销售记录、查询物流状态 |
| **Skill** | 教大模型如何**处理数据** | 会议总结、生成汇报文档     |

**核心区别**：

- **MCP（Model Context Protocol）**：解决"数据从哪来"，连接外部数据源
- **Skill**：解决"数据怎么用"，定义处理逻辑和工作流程

## 9. 快速回答模板

**Q：HTML/CSS/JS 分别用什么缓存策略？**

> HTML 设置短缓存（几分钟），因为内容常变且不带哈希；CSS/JS 设置长缓存（一年），通过构建工具生成哈希文件名，内容变则哈希变，确保更新。

**Q：为什么 Postman 没有跨域问题？**

> 跨域是浏览器的安全策略，Postman 是工具不是浏览器，不执行同源策略。只有浏览器才有跨域限制。

**Q：公共组件和业务组件的区别？**

> 公共组件纯展示、不处理数据、追求独立复用；业务组件服务于特定业务、由公共组件组合而成、追求业务模块化。

**Q：MCP 和 Skill 的区别？**

> MCP 是给大模型提供数据（如查销售记录），解决数据从哪来；Skill 是教大模型如何处理数据（如会议总结），解决数据怎么用。
